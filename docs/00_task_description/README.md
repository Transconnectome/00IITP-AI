# 과제 설명서(공고/RFP) 수집 & 내부 요약

이 폴더는 **IITP 과제 설명서/공고/RFP/제출양식/평가기준**을 모아두고, 팀 내부에서 **요구사항을 “해석 가능한 체크리스트”로 변환**하기 위한 곳입니다.

## 1) 여기에 넣을 것

- 공고문 / RFP / 과제 안내서 (PDF/DOCX/HWP 등)
- 제출 양식(제안서 양식, 표/그림 템플릿, 개인정보/보안/윤리 서류 등)
- 평가항목/배점/심사기준
- 예산 편성/간접비/인건비 기준
- 일정(접수/질의/발표/협약/중간점검 등)

## 3) 체크리스트(요구사항 → 실행 항목)

- **제안서 분량/서식**: 글꼴/줄간격/페이지 제한/그림 해상도/서명 여부
- **필수 섹션**: 목표/필요성/차별성/연구내용/WP/추진체계/예산/파급효과/리스크
- **정량 지표**: KPI/마일스톤/TRL/성능·효율 목표
- **윤리·보안·데이터**: IRB, 개인정보/민감정보, 데이터 반출/보관, 보안 등급
- **기술 범위 제약**: 제외/권장 기술, 필수 데이터/플랫폼, 표준/상호운용
- **평가 매핑**: “우리 주장/근거/실행”이 평가항목에 어떻게 대응되는지

## 4) 산출물로 전환하기

설명서를 읽고 나면 아래 문서들을 업데이트:

- `docs/01_project_planning/`: WP/일정/리스크/KPI를 공고 요구에 맞게 확정
- `docs/03_proposal/`: 제안서 아웃라인과 섹션별 초안을 “평가항목 중심”으로 재배치
