# 2026-인공지능-014 (세부5) — 데이터 의미화/표현 연구계획 초안

근거: `docs/00_task_description/parsed/2-1_인공지능.txt`  
타깃 과제: **관리번호 2026-인공지능-014 (품목공모형, 병렬형 세부)**  
세부과제명: **인공지능을 위한 인간 인지 기반의 데이터 의미화 및 표현 기술 개발**

---

## 0) 한 문장 컨셉

> 인간의 **주의(attention)·목표(goal)·맥락(context)** 기반 데이터 처리 원리를 모사해, “더 많이”가 아니라 **“더 의미 있게”** 데이터를 선택·조직·일반화하는 **인지형 데이터 의미화 프레임워크**를 개발한다.

---

## 1) 과제 요구사항을 제안서 요구로 번역

- **문제정의(설명서 톤)**: 무차별 수집/정제 중심의 데이터 파이프라인을 넘어, **의미 기반(semantic) 조직화·일반화·활용**이 가능한 데이터 기술
- **핵심 기능(AS-IS → TO-BE)**:
  - **인지적 수집**: 목표/주의 기반 “필요한 것만” 선택적으로 취득
  - **의미적 통합**: 시공간·개념 관계를 반영한 **의미 연결망(semantic linkage)** 구축
  - **추상화/일반화**: 반복 패턴에서 개념 단위를 뽑아 **요약·압축·일반화**
  - **동적 관리**: 가치/맥락/활용 빈도 기반 **저장·갱신·망각**
  - **평가/벤치마크**: 위 기능을 정량 평가하는 지표/벤치마크 개발
- **운영 조건**:
  - 010(총괄)과 **연계 수행**, 세부과제 간 **데이터셋 연계/통합**
  - **다학제 공동연구 필수**(뇌/인지/심리/언어 등)
  - 공개SW/기술료/연구데이터 공개 등 **오픈 정책** 요구(세부 조건은 제안서에 명확히 표기)

---

## 2) 연구 질문(제안서형)

1. 데이터 “양”이 아니라 **의미 기반 선택/조직/일반화**가 성능·효율·신뢰성에 어떤 이득을 주는가?
2. 의미 연결망을 어떤 표현(그래프/하이퍼그래프/계층 구조/프로그램 구조 등)으로 만들면 **전이/재사용**이 가능한가?
3. 데이터의 “가치(value)”를 목표/맥락/사용 빈도 관점에서 어떻게 측정하고, **동적 갱신/망각 정책**으로 연결할 것인가?

---

## 3) 핵심 산출물(제안서에 박아야 하는 것)

- **A. 인지형 데이터 의미화 프레임워크**(라이브러리/레퍼런스 구현)
  - Attention/goal-driven acquisition 모듈
  - Semantic linkage(의미 연결망) 구축 모듈
  - Abstraction/generalization(요약·압축·개념화) 모듈
  - Dynamic curation(갱신/망각) 모듈

- **B. 벤치마크 + 지표 세트**
  - “무차별 수집/정제” 대비 **성능 유지/향상 vs 데이터/자원 절감**을 동시에 보여주는 평가

- **C. (가능 범위) 공개 데이터/프로토콜**
  - 총괄(010) 통합 벤치마크/데이터셋과 **연계 가능한 포맷/스키마**

---

## 4) WP 설계(014 중심, 총괄 연계 포함)

- **WP0 (연계/표준)**: 총괄(010) 통합 시나리오/데이터 스키마 협의, 공개 범위 정의, 실험 프로토콜 표준화

- **WP1 (인지적 수집)**: 목표/주의 기반 데이터 선택·취득 메커니즘
  - 출력: selection policy + 측정지표(중복 제거율/정보가치/다운스트림 성능)

- **WP2 (의미 연결망)**: 시공간·개념 관계를 반영하는 semantic linkage 구축
  - 출력: linkage representation + 품질 지표(정합성/일관성/재사용성)

- **WP3 (추상화/일반화)**: 패턴→개념 단위 압축·요약·일반화
  - 출력: abstraction operator + 일반화 성능(전이/zero-shot/robustness)

- **WP4 (동적 관리/망각)**: 가치·맥락 기반 저장/갱신/망각 정책
  - 출력: curation policy + 장기 운영 지표(드리프트/노후화/오용 방지)

- **WP5 (벤치마크/공개SW/확산)**: 벤치마크, 공개SW, 연구데이터 공개 패키징
  - 출력: benchmark v1/v2, leaderboard(가능시), 공개SW 릴리즈

---

## 5) KPI 초안(정량화 제안)

제안서에서 “우리만의 숫자”로 보이게 만드는 최소 KPI 세트:

- **데이터 절감**: 동일 성능 대비 데이터량/토큰/샘플 **감소율**
- **의미 연결망 품질**: linkage consistency/coverage, downstream task에서의 재사용률
- **일반화/전이**: 새로운 도메인/목표로 전환 시 성능 유지(transfer)
- **동적 갱신**: 데이터 드리프트 상황에서 성능 회복 시간/필요 데이터량
- **신뢰성/설명가능성(선택)**: 의미 단위 근거 제시율(근거 그래프/링크 제공)

## 6) Connectome/인지 연구와의 연결(제안서 차별화 포인트)

014는 “데이터 기술”이지만, **인지과학적 모델(주의/목표/맥락)**을 엄밀히 정의하고 실험 프로토콜로 검증하면 차별성이 생깁니다.

- 인지 실험/인지과학 문헌을 “데이터 선택/조직/망각 정책”의 가정으로 명시
- (가능 시) **뇌/커넥톰 기반 네트워크 관점**을 의미 연결망 설계의 정당화로 활용
