# Project Planning (기획)

이 폴더는 **설명서 요구사항을 연구 기획으로 변환**하는 곳입니다.  
핵심 원칙: **평가항목 → 주장(What) → 근거(Why) → 실행계획(How) → 검증(Proof)**.

## Recommended order

1. `01_vision.md`: 한 문장 비전 + 1페이지 컨셉(왜 지금, 왜 우리, 왜 IITP)
2. `02_research_questions.md`: 핵심 가설/연구질문/성공지표
3. `03_work_packages.md`: WP(세부과업) 구조화 + 산출물/마일스톤
4. `04_timeline.md`: 연차별 일정/게이트/의사결정 포인트
5. `05_risks_and_ethics.md`: 기술/운영/데이터/윤리 리스크 및 대응

## Templates

- WP 템플릿: `templates/work_package.md`
- KPI/평가 매핑: `templates/kpi_mapping.md`
- 리스크 레지스터: `templates/risk_register.md`
