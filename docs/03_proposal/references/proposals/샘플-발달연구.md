- **목표 → 세부목표가 ‘데이터 파이프라인’ 그 자체**:
  - 데이터 탐색/수집 → IRB/승인 → 정제/표준화 → 품질관리 → 공유/카탈로그 → 모델링/검증
- **플랫폼/카탈로그 강조**:
  - 연구 결과 “문서”가 아니라, 데이터·도구·카탈로그를 산출물로 명시
- **불확실성/연합학습/샘플 효율** 같은 ‘현장형’ 이슈를 연구내용에 포함
